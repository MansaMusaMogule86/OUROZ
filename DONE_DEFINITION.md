# \# DONE\_DEFINITION.md

# 

# A task is only done when every section below is true.

# 

# \## Functional

# 1\. The feature or fix works in the real intended flow

# 2\. Real data is used where required

# 3\. Failure paths are handled

# 4\. Permissions and roles behave correctly

# 5\. No critical regression is introduced in adjacent flows

# 

# \## Technical

# 1\. Types are clean for touched files

# 2\. Imports are clean

# 3\. No obvious runtime errors are introduced

# 4\. No broken route or contract is introduced

# 5\. No secret leakage to client code exists

# 6\. Server and client boundaries remain correct

# 

# \## UX

# 1\. Loading state exists where needed

# 2\. Empty state exists where needed

# 3\. Error state exists where needed

# 4\. Success feedback exists where appropriate

# 5\. UI remains visually consistent with design\_standards.md

# 

# \## Verification

# 1\. Touched flows were checked

# 2\. Manual test notes are provided

# 3\. Remaining risks are stated honestly

# 4\. The task does not violate QA\_RELEASE\_CHECKLIST.md
